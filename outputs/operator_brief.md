# Operator Brief: Cumbuca

Cumbuca gets a local, deterministic pressure test around cumbuca, wedge, and proprietary. The useful part is the repeatable evidence path from fixture to failure to operator action.

## Highest-leverage checks

- cumbuca evidence replay -> block release until cited evidence is regenerated (cumbuca_coverage, evidence ev_0044).
- abstractions operator packet -> accept only if decision claims cite fixture evidence (wedge_risk, evidence ev_0011).
- proprietary regression harness -> open a regression issue with trace and benchmark delta (proprietary_precision, evidence ev_0022).
- wedge boundary probe -> route to reviewer with evidence packet (abstractions_latency, evidence ev_0077).

## What makes this useful

The workflow is intentionally local and deterministic. A reviewer can run the same fixture set, inspect the evidence IDs, open the dashboard, and see exactly why a recommendation passed, went to review, or blocked.
